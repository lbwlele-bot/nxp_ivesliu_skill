# to_absorb

这里放待归纳的高价值信息。

用途：

- 暂存执行实际任务时新出现的、可复用的、有长期价值的信息
- 先保留原始上下文，再由用户后续决定怎么吸收到系统

放进来的内容至少要能回答：

- 这条信息来自哪个 case
- 支撑这条信息的证据或原始线索是什么
- 它沉淀出的可复用结论是什么
- 它后续可能该吸收到哪一层

推荐最小模板：

```markdown
# <日期-主题>

## 来源 case

- `<work/<case>>`

## 证据

- 记录原始日志、命令输出、文件路径或人工确认来源。

## 可复用结论

- 写成后续 agent 可以直接复用的判断。

## 建议吸收层

- `skill` / `support_level` / `memory` / 暂不吸收
```

这里不是 case 工作目录，也不是正式 skill。
完整过程、原始日志、构建产物仍应留在对应 `work/<case>/` 下。
