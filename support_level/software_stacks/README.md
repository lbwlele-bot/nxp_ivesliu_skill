# software_stacks

这里放跨项目的软件栈身份说明。

它不属于单个源码项目，也不属于某一个 release 包。
它负责回答：

- 某个软件栈名称意味着哪些版本家族
- 这条软件线通常影响哪些源码项目、release 包和固定输入
- 这条链路对最终可验证产物有什么结构性影响

当前已有软件线：

- `rte.md`
  Real-Time Edge 相关版本身份、LF 家族和 `flash.bin` 内容差异入口

使用规则：

1. 如果用户说的是 `RTE 3.3`、`RTE 3.4` 这类软件栈身份，先看这里
2. 确认软件栈身份后，再进入 `compile_targets/` 判断最终产物线
3. 真正落到源码项目时，再进入 `code_assets/projects/<project>/USAGE.md`
4. 真正落到不可切版本的发布包时，再进入 `release_packages/`

不要把软件线约束散落到单个源码项目里。
单个项目的 `USAGE.md` 只记录“上层已经判定要走这条软件线以后，
本项目局部应该怎么做”。
