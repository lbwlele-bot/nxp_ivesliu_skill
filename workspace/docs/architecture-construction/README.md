# architecture-construction

这里保存架构手册和施工手册的版本化文档。

目录名不用 `versions`，因为 `v1`、`v2`、`v3` 已经表达版本；
这里显式说明下面收的是同一组“架构 + 施工”文档。

## 当前版本

- `v3/ARCHITECTURE.md`
  当前推荐架构说明。它是完整独立版本，聚焦第一个实际硬化点：
  分布式 `RELATION.yaml` 关系层。
- `v3/CONSTRUCTION.md`
  当前推荐施工手册。它说明如何在关键目录旁边逐步增加机器可读关系，
  为未来 MCP resolver 提供可靠数据源。

## 历史版本

- `v2/ARCHITECTURE.md`
  v3 前的当前系统架构基线。
- `v2/CONSTRUCTION.md`
  v3 前的施工手册基线。
- `v2/previous/`
  更早的 v2 草稿快照，保留用于回看演进过程。
- `v1/`
  当前仓库没有可确认的完整 v1 文档快照，只保留说明。

## 版本使用原则

- 做当前系统维护，读 `v3/`。
- 回看为什么会走到当前设计，读 `v2/` 和 `v2/previous/`。
- 不从旧版本文档里直接复制规则进当前系统；需要先按当前目录和真实 case 重新核对。
